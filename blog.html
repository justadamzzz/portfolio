Student Life 3 June 2026

My Journey from Science Stream to Software Development

Before entering Computer Science, my academic background was mainly focused on science subjects. Starting software development was challenging, but it became a turning point in my growth.

I did not enter my degree with a strong Computer Science background. Before UniSZA, most of my learning was built around science subjects, formulas, experiments, and theory. When I started Software Development, I had to face a very different kind of thinking. Programming, databases, system design, and technical problem-solving were new areas that required patience and a lot of practice.

My first semester result was a GPA of 3.54. To be honest, I felt disappointed at first because I knew I could do better. At the same time, that result made me realize that Computer Science was not something I could understand by only reading notes before exams. I needed to write code, make mistakes, debug properly, and manage my time with more discipline.

After that semester, I started changing the way I studied. I practiced coding more consistently, asked questions when I was stuck, and tried to understand the logic behind each assignment instead of only finishing it. Slowly, I became more confident with programming and software development. Seeing my results improve in the following semesters reminded me that progress is possible when I stay consistent. Today, I am more motivated to continue growing toward a future career as a Software Engineer, Data Engineer, or AI Engineer.

Artificial Intelligence 5 June 2026

Why I Am Interested in Artificial Intelligence

Artificial Intelligence interests me because it combines logic, data, automation, and problem-solving. It shows how software can become smarter and more helpful.

As a Computer Science student, Artificial Intelligence feels exciting because it connects many areas that I want to understand better. AI is not only about making a system look advanced. It is about using data, logic, and algorithms to help solve problems in a more meaningful way.

I like how AI can support decision-making, analyze patterns, automate repeated tasks, and improve user experience. Even a simple chatbot can become useful when it understands what the user wants and guides them to the right answer. That is one reason I became interested in chatbot systems.

One project that connects with this interest is my BiteBooking Restaurant Booking System. The chatbot in that system uses hybrid NLP and rule-based logic to help users filter food based on their preferences. Building and planning that idea helped me see how AI concepts can make a normal booking system more user-friendly. AI is challenging because it requires continuous learning, but that challenge is also what makes it interesting for me.

Web Development 7 June 2026

Building Interactive Web Projects as a Student

Web development became one of the best ways for me to turn ideas into real working systems, from private systems to interactive creative projects.

Web development is one of the areas where I can clearly see an idea turn into something usable. When I work with HTML, CSS, JavaScript, PHP, MySQL, AJAX, and CodeIgniter, I get to understand how the interface, logic, database, and user flow connect with each other.

Projects such as the RESTful Web Service for Vet Clinic Management helped me understand CRUD operations and how AJAX can make a system feel smoother without refreshing the page. BiteBooking gave me experience with a more user-focused booking flow and chatbot concept. The Bento Box Food Decoration Website also helped me explore interactive UI design and creative DOM interaction.

These projects taught me that a working system is more than just code that runs. It needs clear navigation, understandable actions, good layout, and a flow that makes sense to users. Some of my projects are private because they are academic or coursework systems, but I still show screenshots or project summaries in my portfolio to explain what I learned from them.

IoT Project 9 June 2026

My IoT Mini Goal Scoreboard Project

This project showed me how software and hardware can work together to create a smart mini football goal with automatic scoring and ball return.

My IoT Mini Goal Scoreboard System was a project that made me think beyond normal web pages and software screens. The idea was to build a smart mini football goal where a player can kick a ball into the goal area and the system responds automatically.

The project uses an ESP8266 as the main controller, with an ultrasonic sensor to detect the ball distance. When the ball enters the goal area, the score updates on the OLED display, the RGB LED gives visual feedback, the buzzer plays a goal sound, and the servo motor pushes or returns the ball back to the player. Telegram control also adds another layer by allowing score notifications and commands such as checking or resetting the score.

This project helped me understand IoT system design in a more practical way. I learned how sensor detection, distance measurement, hardware output, and software logic need to work together smoothly. It also reminded me that good prototypes should think about the user. In this case, the ball return feature makes the mini goal more useful because the player can continue practicing without collecting the ball every time.
